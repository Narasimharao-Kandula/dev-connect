================================================================================
  STAGE 1: REQUIREMENT ANALYSIS
================================================================================

  Objective: Identify business needs, define user stories, and finalize
  feature scope for the developer collaboration platform.

  1.1  Identify Target Users
  ──────────────────────────
    Target Audience: Software developers of all skill levels
    User Personas:
      - Junior Developer: Seeks mentorship and project experience
      - Senior Developer: Looks for team members for side projects
      - Hobbyist: Wants to contribute to open-source
      - Startup Founder: Needs co-founders with complementary skills

  1.2  Define Core Business Needs
  ────────────────────────────────
    Need 1: Developer Discovery
      - Search developers by skill, location, availability
      - View detailed developer profiles with collaboration history
      - Follow/unfollow developers

    Need 2: Real-Time Collaboration
      - Instant messaging between developers
      - Team chat rooms for project groups
      - Live notifications for requests and updates

    Need 3: Project Management
      - Create and manage projects with skill requirements
      - Kanban-style task tracking
      - Team member management

    Need 4: Smart Matchmaking
      - Match developers to projects based on skills
      - Suggest team members with complementary skills
      - Automated collaboration score tracking

    Need 5: User Management
      - Email/password registration with email verification
      - OAuth login via Google and GitHub
      - Password reset flow
      - Profile management with skills

  1.3  Define Functional Requirements
  ────────────────────────────────────
    FR-01: User registration with name, email, and password
    FR-02: Email verification via token
    FR-03: Login with email/password or OAuth (Google, GitHub)
    FR-04: Password reset via email link
    FR-05: Profile creation with bio, skills, experience, avatar
    FR-06: Full-text search across users and projects
    FR-07: Skill-based filtering
    FR-08: Project CRUD with skill tagging
    FR-09: Kanban task board with drag-and-drop
    FR-10: Team creation and member management
    FR-11: 1:1 real-time chat with typing indicators
    FR-12: Team group chat rooms
    FR-13: Send and manage collaboration requests
    FR-14: Real-time notifications
    FR-15: Follow/unfollow developers
    FR-16: Rate and review collaborators
    FR-17: Bookmark projects
    FR-18: Admin dashboard with platform stats
    FR-19: Dark/light theme
    FR-20: Internationalization (4 languages)

  1.4  Define Non-Functional Requirements
  ──────────────────────────────────────
    NFR-01: API response times under 500ms for 95% of requests
    NFR-02: WebSocket message delivery under 100ms
    NFR-03: Support concurrent WebSocket connections
    NFR-04: Rate limiting on auth endpoints (10 req/15min)
    NFR-05: Password hashing with bcrypt (12 rounds)
    NFR-06: Input sanitization against XSS
    NFR-07: JWT-based stateless authentication
    NFR-08: Responsive design supporting mobile and desktop
    NFR-09: Dark and light theme support
    NFR-10: Test coverage >90% for critical paths

    - 4 core business needs
    - 20 functional requirements (FR-01 to FR-20)

  2.4  Design Real-Time Events (Socket.IO)
  ─────────────────────────────────────────

    Client → Server:
      chat:send         { conversationId, content, attachmentUrl? }

      chat:stop-typing  { conversationId }
      messages:read     { conversationId, messageIds }
      team:send         { teamId, content }

    Server → Client:
      chat:receive        { id, conversationId, senderId, content, createdAt }
      chat:typing         { conversationId, userId }
      chat:stop-typing    { conversationId, userId }
      messages:read       { conversationId, userId, messageIds }
      team:receive        { id, teamId, senderId, content, createdAt }
      notification:new    { id, type, message, metadata }
      user:online         { userId }
      user:offline        { userId }

    Rooms:
      user:{userId}       — Private room for user-specific notifications
      conversation:{id}   — Room for 1:1 chat participants
      team:{teamId}       — Room for team chat members

  3.6  Implement Socket.IO Server
  ────────────────────────────────
    File: server/src/socket.ts
    ```typescript
    import { Server as HTTPServer } from "http";
    import { Server } from "socket.io";
    import jwt from "jsonwebtoken";
    import { env } from "./config/env";

    let io: Server;
    const onlineUsers = new Set<string>();

    export function getIO(): Server { return io; }
    export function isUserOnline(userId: string): boolean {
      return onlineUsers.has(userId);
    }

    export function setupSocket(httpServer: HTTPServer) {
      io = new Server(httpServer, {
        cors: { origin: env.CLIENT_URL.split(",").map(s => s.trim()), credentials: true },
      });

      io.use((socket, next) => {
        const token = socket.handshake.auth.token;
        const decoded = jwt.verify(token, env.JWT_SECRET) as { userId: string };
        socket.data.userId = decoded.userId;
        onlineUsers.add(decoded.userId);
        next();
      });

      io.on("connection", (socket) => {
        socket.join(`user:${socket.data.userId}`);
        socket.broadcast.emit("user:online", socket.data.userId);

        socket.on("chat:send", async (data) => {
          // Persist message, emit to conversation room
        });

        socket.on("chat:typing", (data) => {
          socket.to(`conversation:${data.conversationId}`)
            .emit("chat:typing", { conversationId: data.conversationId, userId: socket.data.userId });
        });

        socket.on("disconnect", () => {
          onlineUsers.delete(socket.data.userId);
          socket.broadcast.emit("user:offline", socket.data.userId);
        });
      });
    }
    ```

  3.10 Set Up Authentication Store (Zustand)
  ───────────────────────────────────────────
    File: client/src/store/authStore.ts
    ```typescript
    import { create } from 'zustand';
    import api from '../api/client';
    import { connectSocket, disconnectSocket } from '../api/socket';

    interface AuthState {
      user: User | null;
      token: string | null;
      loading: boolean;
      error: string | null;

      login: (email: string, password: string) => Promise<void>;
      register: (name: string, email: string, password: string) => Promise<void>;
      logout: () => void;
      loadUser: () => Promise<void>;
    }

    export const useAuthStore = create<AuthState>((set, get) => ({
      user: null,
      token: localStorage.getItem('token'),
      loading: false,
      error: null,

      login: async (email, password) => {
        set({ loading: true, error: null });
        try {
          const { data } = await api.post('/auth/login', { email, password });
          localStorage.setItem('token', data.token);
          connectSocket(data.token);
          set({ user: data.user, token: data.token, loading: false });
        } catch (err: any) {
          set({ error: err.response?.data?.error || 'Login failed', loading: false });
        }
      },

      register: async (name, email, password) => {
        set({ loading: true, error: null });
        try {
          const { data } = await api.post('/auth/register', { name, email, password });
          localStorage.setItem('token', data.token);
          connectSocket(data.token);
          set({ user: data.user, token: data.token, loading: false });
        } catch (err: any) {
          set({ error: err.response?.data?.error || 'Registration failed', loading: false });
        }
      },

      logout: () => {
        localStorage.removeItem('token');
        disconnectSocket();
        set({ user: null, token: null });
      },

      loadUser: async () => {
        const token = get().token;
        if (!token) {
          set({ loading: false });
          return;
        }
        try {
          const { data } = await api.get('/auth/me');
          connectSocket(token);
          set({ user: data, loading: false });
        } catch {
          localStorage.removeItem('token');
          set({ user: null, token: null, loading: false });
        }
      },
    }));
    ```

    Expected Result:
    - User can register and receive JWT token
    - User can login and token is stored in localStorage
    - Auth state persists across page refreshes (loadUser on mount)

  5.4  Create Deploy Pipeline (GitHub Pages)
  ──────────────────────────────────────────
    File: .github/workflows/deploy.yml
    ```yaml
    name: Deploy Client to GitHub Pages

    on:
      push:
        branches: [main, master]
      workflow_dispatch:

    permissions:
      contents: read
      pages: write
      id-token: write

    concurrency:
      group: pages
      cancel-in-progress: true

    jobs:
      build:
        name: Build Client
        runs-on: ubuntu-latest
        defaults:
          run:
            working-directory: client

        steps:
          - uses: actions/checkout@v4
          - uses: actions/setup-node@v4
            with:
              node-version: 22
              cache: npm
              cache-dependency-path: client/package-lock.json

          - name: Configure Pages
            uses: actions/configure-pages@v5

          - name: Derive base path
            id: base
            run: echo "path=/$(echo ${{ github.repository }} | cut -d'/' -f2)/" >> $GITHUB_OUTPUT

          - name: Install dependencies
            run: npm ci

          - name: TypeScript check
            run: npx tsc -b

          - name: Build with custom base
            run: npm run build -- --base=${{ steps.configure-pages.outputs.base_path || steps.base.outputs.path }}
            env:
              VITE_API_URL: ${{ secrets.VITE_API_URL || 'https://dev-connect-api.onrender.com' }}

          - name: Copy 404.html for SPA routing
            run: cp dist/index.html dist/404.html

          - name: Upload Pages artifact
            uses: actions/upload-pages-artifact@v3
            with:
              path: client/dist

      deploy:
        name: Deploy to GitHub Pages
        needs: build
        runs-on: ubuntu-latest
        environment:
          name: github-pages
          url: ${{ steps.deployment.outputs.page_url }}

        steps:
          - name: Deploy
            id: deployment
            uses: actions/deploy-pages@v4
    ```
